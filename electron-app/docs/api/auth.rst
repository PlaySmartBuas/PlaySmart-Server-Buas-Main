Auth API
========

Endpoints
---------

- ``POST /api/auth/signup`` — Register a new user.
  - Request body: ``UserCreate`` model (username, email, password, etc.).
  - Response: created ``User`` object (HTTP 201).

- ``POST /api/auth/login`` — Authenticate and receive an access token.
  - Request body: ``UserLogin`` (username and password).
  - Response: ``Token`` object containing ``access_token`` and ``user`` info.

Notes
-----
Auth endpoints are implemented in ``backend/app/router/auth.py`` and rely on the database and utility functions in ``app.utils.auth``. Tokens are bearer JWTs generated by the backend.

Example (curl)
--------------

Create user:

.. code-block:: bash

   curl -X POST http://localhost:8000/api/auth/signup \
     -H "Content-Type: application/json" \
     -d '{"username":"alice","email":"alice@example.com","password":"secret"}'

Login:

.. code-block:: bash

   curl -X POST http://localhost:8000/api/auth/login \
     -H "Content-Type: application/json" \
     -d '{"username":"alice","password":"secret"}'

Axios example (login)
----------------------

.. code-block:: javascript

   import axios from 'axios';

   axios.post('http://localhost:8000/api/auth/login', { username: 'alice', password: 'secret' })
     .then(res => console.log('token', res.data))
     .catch(err => console.error(err.response?.data || err.message));

Example request/response models
-------------------------------

Request (signup) - ``UserCreate`` schema

.. code-block:: json

   {
     "username": "alice",
     "email": "alice@example.com",
     "role": "player",
     "team": "TeamA",
     "password": "secret"
   }

Response (token) - ``Token`` schema

.. code-block:: json

   {
     "access_token": "<jwt-token>",
     "token_type": "bearer",
     "user": {
       "id": 1,
       "username": "alice",
       "email": "alice@example.com",
       "role": "player",
       "team": "TeamA",
       "created_at": "2026-01-01T12:00:00"
     }
   }
